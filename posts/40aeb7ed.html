1.1. Java就业概况与体系

[!note] 解释

  • Java SE（Standard Edition / 标准版）： 是 Java 平台的标准版，包含了 Java 语言最核心、最基础的类库和运行环境（它包括变量、循环、面向对象、集合、IO、多线程、网络编程等核心内容）。

我的理解：Java SE 就像盖楼的地基——地基不稳，什么方向都走不远。先把 SE 学透，其他方向的学习曲线会大幅降低。

2.3. Java运行机制及运行过程

2.3.1. 解释java重要的跨平台特点

跨平台性原理解释（一次编译，到处运行）：

  1. 玩Java的前提是有一个java编写的.java文件，这个叫源文件。
  2. 通过javac进行编译，结果为.class字节码文件。
  3. 字节码文件交由不同操作系统上的 JVM (Java虚拟机) 来执行。Windows 有 Windows版本的 JVM，Linux 有 Linux版本的 JVM，Mac 有 Mac版本的 JVM。
  4. JVM是一个虚拟的计算机，具有指令集并使用不同的存储区域。负责执行指令,管理数据、内存、寄存器，包含在JDK中.

    我的理解： Java程序本身不跨平台，对于不同的平台，有不同的虚拟机。Java虚拟机机制屏蔽了底层运行平台的差别,帮助我们实现了“一次编译，到处运行”

2.4. 什么是JDK，JRE

2.4.1. JDK基本介绍

  1. JDK的全称Java Development Kit，中文名为“Java开发工具包”。JDK整体来看有两部分内容：
    JDK = JRE + java的开发工具[java，javac，javadoc，javap等]
  2. JDK 是提供给 Java开发人员使用的，其中包含了java 的开发工具，也包含了用于运行.class文件的JRE工具。对于只运行开发好的 .class 文件的人，只需要JRE即可，不过安装了 JDK，就不用在单独安装 JRE 了，所以一般直接安装JDK。

2.4.2. JRE基本介绍

  1. JRE的全称Java Runtime Environment，中文名为“Java 运行环境”。JRE整体来看也是两部分内容：
    JRE = JVM + Java 的核心类库[类]
  2. 包括 Java 虛拟机(JVM Java Virtual Machine)和 Java 程序所需的核心类库等，如果想要运行一个开发好的 Java 程序,计算机中只需要安装 JRE 即可。

2.4.3. JDK、JRE和JVM的包含关系

  1. JDK = JRE+ 开发工具集(例如 Javac,java 编译工具等）

  2. JRE = JVM + Java SE 标准类库(java 核心类库)

  3. 如果只想运行开发好的 .class 文件 只需要 JRE

2.7. Java开发注意事项和细节说明

注意事项与说明
1. Java源文件以 .java 为扩展名。源文件的基本组成部分是类(class)，如本类中的Hello类。
2. 如果源文件包含一个public类，则文件名必须按该类名命名!
3. Java应用程序的执行入口是main()方法。它有固定的书写格式:public static void main(String[] args) {…}
4. Java语言严格区分大小写。
5. Java方法由一条条语句构成，每个语句以“;”结束。
6. 大括号都是成对出现的，缺一不可。[习惯，先写 {}再写代码]
7. 一个源文件中最多只能有一个public类。 其它类的个数不限。也可以将main方法写在非public类中，然后指定运行非public类，这样入口方法就是非public的main方法
8. 如果源文件中没有公共类（public class），文件名可自行拟定。

⚠️ 易踩坑：public 类名和文件名不一致是初学者最常见的错误，编译时会直接报错。

第一步：明确需求 学一门新技术，首先要知道为什么学。常见驱动力：

  • 跳槽/求职要求

  • 能解决但不完美 → 学习动力更足
  • 完全解决不了 → 压力驱动，更要学
  • 好处：帮你理解新技术的价值所在，用的时候才能用到精华

第三步：引出并确认要掌握的新技术 明确”我要学的到底是哪个”，必要时做横向对比（如用 Redis 还是自己写缓存）

第四步：只学基本原理 + 基本语法，不考虑细节

  • 先建立整体框架
  • 上来就抠细节 = 容易陷进去出不来

  • 能跑起来增删改查，工作上的基本需求就能应付

  • 注意事项 / 使用细节 / 使用规范 / 性能优化
  • 这是永无止境的螺旋递增过程

3.15.1. 自动类型转换

√ 介绍
当java程序在进行赋值或者运算时，精度小的类型自动转换为精度大的数据类型，这个就是自动类型转换。
√数据类型按精度(容量)大小排序为(背，规则)

3.15.2. 自动类型转换注意和细节

  1. 有多种类型的数据混合运算时，系统首先自动将所有数据转换成容量最大的那种数据类型，然后再进行计算。

  2. 当我们把精度(容量)大 的数据类型赋值给精度(容量)小 的数据类型时，就会报错，反之就会进行自动类型转换。

  3. (byte,short)和 char之间不会相互自动转换。

  4. byte，short，char 他们三者可以计算，但是计算时它们首先转换为int类型，这意味着计算结果不可以赋值给精度小于int的变量。

  5. boolean 类型不参与转换，也就是说基本数据类型中只有这1种不参与。

  6. 自动提升原则: 表达式结果的类型自动提升为 操作数中最大的类型

3.15.3. 强制类型转换

√介绍
自动类型转换的逆过程，将容量大的数据类型转换为容量小的数据类型。使用时要加上强制转换符 ( )，但可能造成精度降低或溢出,格外要注意

冒泡排序（Bubble Sorting）的基本思想是：通过对待排序序列从后向前（从下标较大的元素开始），依次比较相邻元素的值，若发现逆序则交换，使值较大的元素逐渐从前移向后部，就象水底下的气泡一样逐渐向上冒。
